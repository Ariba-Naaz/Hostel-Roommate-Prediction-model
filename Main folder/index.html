Pipeline Overview
Seven stages from raw Google Form responses to ranked roommate pairs.
1
Load & Clean
CSV loaded. Every student identified by name + roll_no — no anonymous IDs. Jain variants normalised. music_bother string parsed. Missing values: mode for categorical, median for numeric.
2
Encode & Scale
Ordinal features → rank integers → MinMaxScaler to [0,1]. Food and study environment → one-hot. All fitted encoders saved and reused for new-user scoring — no data leakage.
3
PCA (90% variance)
Dimensionality reduced separately per gender. PCA is used only for cosine shortlisting and visualisation — NOT for the compatibility score, which runs on original scaled features.
4
4-Model Clustering
KMeans DBSCAN GMM Agglomerative all run per gender. Winner = highest silhouette score on the original feature space. DBSCAN noise points assigned to nearest centroid.
5
Cosine Shortlisting
Within each gender × cluster bucket, cosine similarity computed for all student pairs. Top-10 candidates per student shortlisted — fast O(n) pre-filter.
6
Compatibility Scoring
Each shortlisted pair scored via total_compatibility(). Hard filters: smoke=0 or food<0.2 → score=0. Ordinal sim = 1−|a−b| (features already in [0,1]). Custom asymmetric matrices for food and smoking. Gender-specific weights.
7
Explanation & Storage
Every pair gets a full feature breakdown: Matched (≥75%), Partial (35–74%), Conflict (<35%). Top-3 per student saved to disk — batch view loads instantly without re-scoring.
